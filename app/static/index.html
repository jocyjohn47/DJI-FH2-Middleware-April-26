Universal Webhook POC – 控制台 业务接口 POST-only

目标：多输入源先与中间件入站鉴权，通过后才进入队列（沙盒用 Redis Streams 代替 Kafka raw topic）。 平台侧可维护每个 source 的： ①入站鉴权（中间件 Token） + ②FlightHub2 三件套（X-User-Token / x-project-uuid / workflow_uuid）。

A) FlightHub2 三件套维护（按 source 热更新）

Worker 每条消息都会按 source 从 Redis 读取配置。Token 刷新/项目关闭/workflow 时效导致失效时，只需在此更新并保存。

C) FlightHub2 三件套提取器（粘贴即可）

支持粘贴 HTTP headers / curl 命令 / JSON 片段，后端提取后可一键应用到上面的 JSON。
{}

B) 中间件入站鉴权（多输入源 → 中间件）

只有鉴权通过的 webhook 才会进入队列。当前 POC：每个 source 配置一个 static token，由调用方在请求头携带。 默认 Header 名：X-MW-Token。

D) 入站鉴权调用示例

curl -X POST http://127.0.0.1:8000/webhook \
  -H 'Content-Type: application/json' \
  -H 'X-MW-Token: <your-source-token>' \
  -d '{"source":"flighthub2","webhook_event":{...}}'
